Module 4 : Régularisation

Diagnostique​

Structure neuronale de base​

  • Fully Connected
    • Initialisation Kaiming He uniforme
    • bias = False
    • Batch Normalization : α=1\boldsymbol{\alpha} = 1α=1, β=0\boldsymbol{\beta} = 0β=0, m=0\mathbf{m} = 0m=0, s=1\mathbf{s} = 1s=1.
    • Présence de skip connections entre certaines couches.
  • Convolution
    • Initialisation Kaiming He normale
    • bias = False
    • Batch Normalization : α=1\boldsymbol{\alpha} = 1α=1, β=0\boldsymbol{\beta} = 0β=0, m=0\mathbf{m} = 0m=0, s=1\mathbf{s} = 1s=1.
    • Présence de skip connections entre certaines couches.
    • Max Pooling
    • Global Average Pooling comme dernière méthode

Information à enregistrer​

  • Bien diagnostiquer = accès à toute l'information
    • But : s'assurer que tout se passe comme voulu
  • Information de performance
    • Valeur de la fonction de coût (loss)
      • Peut voir s'il y a des NaN, inf, si ça augmente au lieu de diminuer, etc.
    • Valeur de la métrique de performance
      • Donne une meilleure idée de la performance que la fonction de coût
  • Information sur le réseau
    • Afficher le nombre de paramètres
      • np.sum([p.numel() for p in model.parameters()])
      • pour voir si on entraîne la bonne structure de réseau
    • Afficher les prédictions d'une seule batch en début d'époque
      • Peut voir l'évolution des prédictions.
  • Information sur les données
    • Nombre d'exemples d'entraînment, de validation et de test
      • len(train_loader.dataset), len(valid_loader.dataset), len(test_loader.dataset)
    • Ratio des classes
      • Déetcter un déséquilibre de classes
    • Afficher les exemples d'une seule batch en début d'époque
      • Seulement si c'est informatif (e.g. images)
      • Encore plus important si on fait du data augementation
  • Information sur l'optimisation
    • La norme, moyenne, variance des mises-à-jour
    • La norme, moyenne, variance et proportion de nulles des neurones cachées
      • Donne une idée de la sparsité du réseau (du nombre d'unités mortes)
    • Afficher le temps de calcul d'une batch
      • Estimer le temps total de l'apprentissage
    • Valeur des hyperparamètres

Afficher pour expliquer​

  • Réseaux sont souvent un pipeline de traitement d’image
  • Avoir du code instrumenté pour montrer les images/feature maps à chacune des étapes – Pour pouvoir expliquer à n’importe qui comment votre réseau fonctionne
    • Faire des posters/vidéos scientifiques – Sera très utile pour débugger votre code!

Méthode rouleau compresseur​

  • Pour mieux se familiariser avec un nouveau problème d’apprentissage.
  • Procéder graduellement en 3 étapes :
    • Sous-ensemble des données + sous-ensemble des classes.
    • Toutes les données + sous-ensemble des classes.
    • Toutes les données + toutes les classes.

Étape 1 : Sous-ensemble des données + sous-ensemble des classes​

  • But
    • Démontrer que le réseau peut sur-apprendre
    • Si pas bon en train, alors pas bon en validation et test
  • Astuces
    • Pas de régularisation
    • Pas d'augmentation de données
  • Exemples
    • Classification binaire avec 10 exemples par classe

Étape 2 : Toutes les données + sous-ensemble des classes​

  • But
    • Démontrer que le réseau peut généraliser
  • Astuces
    • Ajouter un peu de régularisation
    • Ajouter un peu d'augmentation de données
  • Exemples
    • Classification binaire avec 100 000 exemples par classe

Étape 3 : Toutes les données + toutes les classes​

  • But
    • Débuter l'entraînement normal
  • Astuces
    • Petit réseau →\rightarrow→ gros réseau
    • Horaire d'entraînement ou patience
    • Augmentation de données maximale
  • Exemple
    • Classification 100 classes avec 100 000 exemples par classe

Analyse​

  • But
    • Obtenir une rétrospective après l'apprentissage
  • Permet de
    • Détecter des étiquettes fausses
    • Détecter un mauvais traitement de données
    • Donner une intuition sur la difficulté du problème

Analyse d'erreur​

  • Exemple
    • Aléatoirement, regarder quelques exemples de prédictions correctes et erronées
    • L'exemple ayant la meilleure performance et la pire performance
    • L'exemple ayant la plus grande incertitude (prob 0.5 classification binaire)

Analyse des minibatches​

  • S'assurer que
    • Tous les exemples sont différents
    • Les étiquettes sont correctes
  • Afficher l'augmentation de données
    • Regarder si elle respect l'invariance de classe

1. Pénalités sur les poids​

  • Cherche à limiter la capacité du modèle
  • Ajoute une pénalité à la fonction de coût : J~(θ;X,y)=J(θ;X,y)+λΩ(θ)\tilde{J}(\theta; X, y) = J(\theta;X,y) + \lambda \Omega(\mathbf{\theta})J~(θ;X,y)=J(θ;X,y)+λΩ(θ)
  • Plusieurs choix possibles pour Ω(⋅)\Omega(\cdot)Ω(⋅)
  • Pratique généralisée : pénalités appliquées seulement sur les poids www et non les biais (régulariser les biais aura tendance à underfitter)
  • Possible d'avoir une fonction Ω(⋅)\Omega(\cdot)Ω(⋅) différente pour chaque couche, mais fait exploser le nombre d'hyperparamètres
  • Exemple réseau sigmoïde
    • Poids www faibles rightarrowrightarrowrightarrow dégénère en réseau linéaire
W1 = scale * randn(30, 1)
b1 = scale * randn(30, 1)
W2 = randn(1, 30)
b2 = randn(1, 1)

Pénalités Omega = L2 norm​

  • Pénalité Ω=L2\Omega = \text{L}^2Ω=L2 norm
  • Autre nom : weight decay (valide seulement pour optimisation SGD)
  • La plus commune
  • Ω(θ)=12∣∣w∣∣22=12∑iwi2\Omega(\mathbf{\theta}) = \frac{1}{2} || w ||_2^2 = \frac{1}{2} \sum_{i} w_i^2Ω(θ)=21​∣∣w∣∣22​=21​∑i​wi2​
  • Cherche à rapprocher les poids www vers l'origine.
  • Son gradient, cherche à éroder les poids à chaque itération
    • Affectera peu les dimensions de www qui ont un grand gradient
∇wJ~(w;X,y)=λw+∇wJ(w;X,y)w←w−ϵ∇wJ~(w;X,y)w←(1−ϵλ)w−ϵ(λw+∇wJ(w;X,y))\begin{align*} \nabla_w \tilde{J}(w;X,y) & = \lambda w + \nabla_w J(w;X,y) \\ w & \leftarrow w - \epsilon \nabla_w \tilde{J}(w;X,y) \\ w & \leftarrow (1 - \epsilon \lambda) w - \epsilon (\lambda w + \nabla_w J(w;X,y)) \end{align*}∇w​J~(w;X,y)ww​=λw+∇w​J(w;X,y)←w−ϵ∇w​J~(w;X,y)←(1−ϵλ)w−ϵ(λw+∇w​J(w;X,y))​

Pénalités Omega = L1 norm​

  • Moins commune
  • Ω(θ)=∣∣w∣∣1=∑iwi\Omega(\mathbf{\theta}) = || w ||_1 = \sum_{i} w_iΩ(θ)=∣∣w∣∣1​=∑i​wi​
  • Va induire des solutions sparse (beaucoup de poids à 0) pour λ\lambdaλ suffisamment grand
  • Indique quels features sont utiles (features selection)
  • Le changement de poinds à l'update est indépendant de la valeur de www
J~(θ;X,y)=J(θ;X,y)+λΩ(θ)∇wJ~(w;X,y)=∇wJ(w;X,y)+λsign(w)\begin{align*} \tilde{J}(\theta;X,y) & = J(\theta;X,y) + \lambda \Omega(\theta) \\ \nabla_w \tilde{J}(w;X,y) & = \nabla_w J(w;X,y) + \lambda \text{sign}(w) \end{align*}J~(θ;X,y)∇w​J~(w;X,y)​=J(θ;X,y)+λΩ(θ)=∇w​J(w;X,y)+λsign(w)​

sign(w)={−1 si x<00 si x=01 si x>0\text{sign}(w) = \begin{cases} -1 & \text{ si } x < 0 \\ 0 & \text{ si } x = 0\\ 1 & \text{ si } x > 0 \end{cases}sign(w)=⎩⎨⎧​−101​ si x<0 si x=0 si x>0​

Max norm constraint​

  • Limite sur la norme absolue des poids d'un neurone ∣∣w∣∣2<c||w||_2 < c∣∣w∣∣2​<c
  • Valeur typique de ccc est 3 à 4 (voir ici)
  • Empêche le réseau « d’exploser »
  • Approche moins connue, mais utilisée dans Dropout: A Simple Way to Prevent Neural Networks from Overfitting, Srivastava et al. JMLR 2014.

2. Data augmentation​

  • Le meilleur régularisateur c’est d’avoir plus de données
  • Pas toujours possible d’en collecter plus
  • Créer des fausses données à partir des vraies
  • Fonctionne particulièrement bien pour classification visuelle, car on y cherche l’invariance à certaines transformations : – Rotation – Translation – Réflexion horizontale ou verticale – Échelle – Intensité lumineuse, etc.
  • Idéalement, comprendre le processus (physique) de création des données, pour le calquer
  • Exemples avec des images
    • Horizontal flip
    • Vertical flip
    • Rotation
    • Translation
    • Zoom (random scale)
    • Random crop
    • Random noise
    • Random brightness
    • Random contrast
  • Note pratico-pratique
    • Le data augmentation se fait souvent sur le CPU, conjointement avec lecture disque multithread
    • Rotation est coûteuse, fonction de la taille d’image
      • Faites alors : crop rightarrowrightarrowrightarrow rotation rightarrowrightarrowrightarrow crop
    • Attention aux effets de bord des rotations
      • Artéfacts nuisibles mais le réseau peut apprendre à les ignorer
    • Versions rapides : pillow-simd, albumentation
    • Doit estimer la fourchette des paramètres de transformation (e.g. angle entre ±10o) – encore plus d’hyperparamètres à identifier – RandAugment utilise un petit espace de recherche
    • Voir Ekin D. Cubuk, Barret Zoph, Jonathon Shlens, Quoc V. Le, RandAugment: Practical automated data augmentation with a reduced search space, CVPR 2019.
transforms = [
    'Identity', 'AutoContrast', 'Equalize',
    'Rotate', 'Solarize', 'Color', 'Posterize',
    'Contrast', 'Brightness', 'Sharpness',
    'ShearX', 'ShearY', 'TranslateX', 'TranslateY'
]

def randaugment(N, M):
    """
    Generate a set of distortions.

    Args:
        N: Number of augmentation transformations to apply sequentially.
        M: Magnitude for all the transformations.
    """
    sampled_ops = np.random.choice(transforms, N)
    return [(op, M) for op in sampled_ops]

3. Robustesse au bruit​

  • Possible aussi d’ajouter du bruit sur :
    • L’entrée xxx
      • Similarité avec denoising autoencoders
    • Les unités cachées hih_ihi​
      • Simule variabilité des features de plus haut niveau d’abstraction
      • Similarité avec Dropout (dans quelques acétates)
    • Les poids w\mathbf{w}w (moins courant)
      • Indique qu’on cherche un modèle moins sensible aux paramètres (sections plus planes du profil de coût JJJ)
      • Parfois utilisé dans les RNN
      • Sonde la confiance du réseau
  • Bien sélectionner la magnitude du bruit

  • Similaire à l’esprit de PCA (principal component analysis), embedding
