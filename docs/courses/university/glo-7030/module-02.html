Entropie croisée (cross-entropy)​

Distance asymétrique entre deux distributions de probabilités.

Cross-entropy

Cette image explique le concept de la "cross-entropy" (entropie croisée) qui est une mesure souvent utilisée dans les domaines de l'apprentissage automatique et de la classification statistique.

Voici les points clés de cette diapositive :

  1. Choix de p et q : L'entropie croisée est définie pour deux distributions de probabilité p et q, où ppp est la distribution réelle et qqq est la distribution estimée. L'entropie croisée H(p,q)H(p, q)H(p,q) est donnée par la formule H(p,q)=−∑p(x)log⁡q(x)H(p, q) = -\sum p(x) \log q(x)H(p,q)=−∑p(x)logq(x), où la somme est calculée sur toutes les classes ccc.
  2. Exemple de vecteur de probabilités : Sur la gauche, deux vecteurs sont affichés. Le premier vecteur y^=f(x)\hat{y} = f(x)y^​=f(x) représente les probabilités prédites pour chaque classe (par exemple, dans une classification à trois classes). Le deuxième vecteur yyy est un "1-hot vector", ce qui signifie qu'il contient un '1' pour la classe réelle (la vérité de terrain) et '0' pour toutes les autres classes. C'est une représentation standard pour la classe cible dans les problèmes de classification.
  3. Formule simplifiée avec 1-hot vector : La diapositive indique ce qui se passe lorsqu'on utilise un vecteur cible "1-hot" dans le calcul de l'entropie croisée. La formule de l'entropie croisée se simplifie alors en L(y,y^)=−log⁡y^cibleL(y, \hat{y}) = -\log \hat{y}_{cible}L(y,y^​)=−logy^​cible​, où y^cible\hat{y}_{cible}y^​cible​ est la probabilité prédite pour la classe réelle.
  4. Problème avec le log(0) : La diapositive met en évidence un problème potentiel lorsque l'on calcule le log d'une probabilité prédite de '0', ce qui entraîne un logarithme de zéro, noté par −∞-\infty−∞ (moins l'infini), ce qui n'est pas souhaitable. Elle indique également que log⁡(1)=0\log(1) = 0log(1)=0, ce qui est correct et ne pose pas de problème dans le calcul.
  5. Conclusion sur le seul choix possible : Pour éviter le problème du log(0), la diapositive conclut que le seul choix possible est d'utiliser l'entropie croisée comme fonction de perte L(y,y^)L(y, \hat{y})L(y,y^​), et elle réitère la formule simplifiée pour l'entropie croisée dans le contexte d'une cible en 1-hot vector.

Cette image explique comment calculer la perte en utilisant l'entropie croisée dans un problème de classification où les classes cibles sont représentées par des vecteurs 1-hot, et souligne l'importance de traiter correctement les cas où les probabilités prédites sont zéro.

Loss function​

See 2.2 Training neural networks - loss function.

Neural network estimates f(x)cf(\boldsymbol{x})_cf(x)c​ of the conditional probability p(y=c∣x)p(y=c|\boldsymbol{x})p(y=c∣x).

We could maximize the probability of the correct class yyy given x\boldsymbol{x}x.

To frame as minimization, we minimize the negative log-likelihood.

l(f(x),y)=−∑c1(y=c)log⁡f(x)c=−log⁡f(x)yl(f(\boldsymbol{x}), y) = - \sum_c 1_{(y=c)} \log f(\boldsymbol{x})_c = - \log f(\boldsymbol{x})_yl(f(x),y)=−c∑​1(y=c)​logf(x)c​=−logf(x)y​

Here log⁡\loglog is the natural logarithm (ln).
