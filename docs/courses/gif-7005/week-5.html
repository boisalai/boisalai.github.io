5.1 Modèles discrimatifs​

Modèles génératifs et modèles discriminatifs​

  • Modèles génératifs de classement
    • Classement basé sur la vraisemblance (densités de probabilité) hi(x)=log⁡P^(Ci∣x)\\ \text{h}_i(\textbf{x}) = \log \hat{P}(C_i|\textbf{x})hi​(x)=logP^(Ci​∣x).
    • Approches paramétriques (incluant densité-mélange) et non paramétriques.
  • Modèles discriminatifs
    • Philosophie : résoudre uniquement le problème de discrimination, l'estimation des densités est une étape "superflue".
    • Obtenir fonction discriminante hi(x∣ϕi)\text{h}_i(\textbf{x}|\phi_i)hi​(x∣ϕi​) selon une paramétrisation ϕi\phi_iϕi​.
  • "When solving a given problem, try to avoid solving a more general problem as an intermediary step." (Vladimir Vapnik)
    • On n'a pas besoin des probabilités a posteriori (des modèles génératifs) mais seulement la décision (des modèles discriminatifs).

  • Modèle génératif : si P(C1∣x)≥P(C0∣x)P(C_1|x) \ge P(C_0|x)P(C1​∣x)≥P(C0​∣x) alors C1C_1C1​, sinon C0C_0C0​.
  • Modèle discriminatif : si x≥0x \ge 0x≥0 alors C1C_1C1​, sinon C0C_0C0​. C'est plus simple...

Discriminants linéaires​

  • Equation d'un discriminant linéaire
hi(x∣wi,wi,0)=∑j=1Dwi,jxj+wi,0\text{h}_i(\textbf{x}|\textbf{w}_i, w_{i,0}) = \sum_{j=1}^D w_{i,j} x_j + w_{i,0}hi​(x∣wi​,wi,0​)=j=1∑D​wi,j​xj​+wi,0​

où wi,jw_{i,j}wi,j​ est le poids de la classe iii et de la dimension jjj.

  • Modèle à deux classes
    • Une seule équation h(x∣w,w0)\text{h}(\textbf{x}|\textbf{w}, w_{0})h(x∣w,w0​)
    • x\bold{x}x appartient à C1C_1C1​ si h(x)≥0\text{h}(\bold{x}) \ge 0h(x)≥0
    • Autrement (lorsque h(x)<0\text{h}(\bold{x}) < 0h(x)<0) x\bold{x}x appartient à C2C_2C2​.
    • Poids w détermine l'orientation de l'hyperplan séparateur (qui tranche l'espace en deux)
    • Biais w0w_0w0​ détermine la position de l'hyperplan séparateur dans l'espace d'entrée

Norme d'un vecteur

La norme de w\bold{w}w (soit ∥w∥\|\bold{w}\|∥w∥) est la somme des éléments de w\bold{w}w au carré.

∥w∥=w12+w02\|\bold{w}\| = \sqrt{w_1^2 + w_0^2}∥w∥=w12​+w02​​

Par exemple, si w=(3,4)\bold{w} = (3, 4)w=(3,4), la norme ∥w∥\| \bold{w} \|∥w∥ est ∥w∥=32+42=9+16=25=5\| \bold{w} \| = \sqrt{3^2 + 4^2} = \sqrt{9 + 16} = \sqrt{25} = 5∥w∥=32+42​=9+16​=25​=5.

5.2 Perceptron​

Perceptron​

  • Perceptron
    • Proposé en 1957 par Rosenblatt
    • Considéré comme le réseau de neurones le plus simple
    • La classe est assignée selon le signe de la fonction discriminante h(x∣w,w0)\text{h}(\textbf{x}|\textbf{w}, w_{0})h(x∣w,w0​)
h(x∣w,w0)=w⊤x+w0,x∈{C1 si h(x∣w,w0)≥0C2 autrement \text{h}(\textbf{x}|\textbf{w}, w_{0}) = \bold{w}^\top \bold{x} +w_0, \bold{x} \in \begin{cases} C_1 \text{ si } \text{h}(\textbf{x}|\textbf{w}, w_{0}) \ge 0 \\ C_2 \text{ autrement } \end{cases}h(x∣w,w0​)=w⊤x+w0​,x∈{C1​ si h(x∣w,w0​)≥0C2​ autrement ​
  • Optimisation selon le critère de perceptron (rt∈{−1,1}r^t \in \{-1,1\}rt∈{−1,1})
    • EpercpE_{percp}Epercp​ donne l'erreur du perceptron.
    • Epercp(w,w0∣X)E_{percp}(\bold{w},w_0|\mathcal{X})Epercp​(w,w0​∣X) donne l'erreur de w\bold{w}w et w0w_0w0​ sur notre jeu de données X\mathcal{X}X.
Epercp(w,w0∣X)=−∑xt∈Yrth(xt∣w,w0)E_{percp}(\bold{w},w_0|\mathcal{X}) = - \sum_{\bold{x}^t \in \mathcal{Y}} r^t \text{h}(\textbf{x}^t|\textbf{w}, w_{0})Epercp​(w,w0​∣X)=−xt∈Y∑​rth(xt∣w,w0​)
  • Y\mathcal{Y}Y représente les données de X\mathcal{X}X mal classées par h(xt∣w,w0)\text{h}(\textbf{x}^t|\textbf{w}, w_{0})h(xt∣w,w0​)
  • Une donnée est mal classé sur rtr^trt et h(xt∣w,w0)\text{h}(\textbf{x}^t|\textbf{w}, w_{0})h(xt∣w,w0​) sont de signes différents. S'ils ont le même signe, alors c'est le même classement puisque nous avons ici un modèle binaire.
Y={xt∈X ∣ rth(xt∣w,w0)≤0}\mathcal{Y} = \{ \bold{x}^t \in \mathcal{X} \, | \, r^t \text{h}(\textbf{x}^t|\textbf{w}, w_{0}) \le 0 \}Y={xt∈X∣rth(xt∣w,w0​)≤0}

Descente du gradient​

  • Minimisation itérative d'un critère d'erreur E(w,w0∣X)E(\bold{w},w_0|\mathcal{X})E(w,w0​∣X) selon un jeu de données X\mathcal{X}X
{w∗,w0∗}=arg min⁡{w,w0}E(w,w0∣X)\{\bold{w}^*,w_0^*\} = \argmin_{\{\bold{w},w_0\}} E(\bold{w},w_0|\mathcal{X}){w∗,w0∗​}={w,w0​}argmin​E(w,w0​∣X)
  • Résolution par dérivées partielles, ∇wE\nabla_wE∇w​E
∇wE=[∂E∂w0  ∂E∂w1  ∂E∂w2  ⋯  ∂E∂wD]\nabla_wE = \left[ \frac{\partial E}{\partial w_0} \,\, \frac{\partial E}{\partial w_1} \,\, \frac{\partial E}{\partial w_2} \,\, \cdots \,\, \frac{\partial E}{\partial w_D} \right]∇w​E=[∂w0​∂E​∂w1​∂E​∂w2​∂E​⋯∂wD​∂E​]
  • Modification des poids wiw_iwi​ dans la direction opposée au gradient (on descend le gradient)
wi=wi+Δwi,Δwi=−η∂E∂wi,i=0,…,Dw_i = w_i + \Delta w_i, \hspace{7pt} \Delta w_i = - \eta \frac{\partial E}{\partial w_i}, \hspace{7pt} i = 0,\ldots,Dwi​=wi​+Δwi​,Δwi​=−η∂wi​∂E​,i=0,…,D
  • η∈[0,1]\eta \in [0,1]η∈[0,1] est le pas ou taux d'apprentissage (learning rate).

  • Descente du gradient wi=wi+Δwi,i=0,…,Dw_i = w_i + \Delta w_i, \hspace{7pt} i = 0,\ldots,Dwi​=wi​+Δwi​,i=0,…,D

Algorithme du perceptron​

  • Étape 1 : Initialiser les poids w\bold{w}w et w0w_0w0​ arbitrairement

Convergence du perceptron​

  • Convergence sur données linéairement séparables
    • Preuve mathématique de convergence existe pour données linéairement séparables
    • Convergence vers une position quelconque du discriminant qui sépare les données
    • Pour données non linéairement séparables, pas de convergence
  • Critère d'erreur en faible lien avec la nature des erreurs

Régression pour le classement​

  • En régression, l'information sur les erreurs est riche

Convergence de la méthode des moindres carrés​

  • Convergence sur données linéairement séparables
    • Placement de l'hyperplan séparateur à une position minimisant l'erreur quadratique
    • Emphase sur les données avec l'erreur la plus grande (les grandes erreurs au carré! pèsent sur la descente du gradient)
    • Forte influence des données bien classées, mais éloignées de l'hyperplan séparateur
  • Données non linéairement séparables
    • Meilleur placement possible de l'hyperplan selon l'erreur quadratique

5.8 Apprentissage en ligne​

Apprentissage en ligne et par lots​

  • Apprentissage par lots
    • Correction des poids une fois à chaque itération, en calculant l'erreur pour tout le jeu de données
    • Relative stabilité de l’apprentissage
  • Apprentissage en ligne
    • Correction des poids pour chaque présentation de données, donc NNN corrections de poids par itération
    • Guidé par l'erreur sur chaque observation (E(w,w0∣xt)E(\bold{w},w_0|\bold{x}^t)E(w,w0​∣xt))
    • Requiert la permutation de l'ordre de traitement à chaque itération pour éviter les mauvaises séquences
    • Apprentissage en ligne est plus rapide que par lots, mais avec risque de plus grandes instabilités

  • Descente du gradient stochastique
    • Aller plus loin que l'apprentissage en ligne : échantillonnage aléatoire avec remise de chaque donnée d’entraînement
    • Algorithme typique :
      1. Échantillonner aléatoirement (uniformément) une observation xt\bold{x}^txt dans X,t∼U(1,N)\mathcal{X}, t \sim \mathcal{U}(1,N)X,t∼U(1,N)
      2. Déterminer la valeur du taux d'apprentissage, typiquement ηl=1/l\eta^l = 1/lηl=1/l où lll est l'indice de la donnée actuelle selon son ordre de traitement
      3. Corriger les poids par descente du gradient Δwj=−ηl∂E(w,w0∣xt)∂wj,j=0,…,D\\ \Delta w_j = - \eta^l \frac{\partial E(\bold{w},w_0|\bold{x}^t)}{\partial w_j}, \hspace{7pt} j=0,\ldots,DΔwj​=−ηl∂wj​∂E(w,w0​∣xt)​,j=0,…,D
      4. Répéter jusqu'à convergence ou épuisement des ressources
  • Requiert un ajustement décroissant du taux d'apprentissage pour chaque donnée
  • Intéressant pour traiter très grands jeux de données, en une pause
  • Permet aussi un arrêt de l'apprentissage à tout moment
  • Peut également être adapté au traitement de flux de données

5.9 Fonctions de base​

Problème du XOR​

  • Exemple de données non linéairement séparables
x1=[0  0]⊤r1=0x2=[0  1]⊤r2=1x3=[1  0]⊤r3=1x4=[1  1]⊤r4=0\begin{matrix} \bold{x}_1 = [0 \,\, 0]^\top & r_1 = 0 \\ \bold{x}_2 = [0 \,\, 1]^\top & r_2 = 1 \\ \bold{x}_3 = [1 \,\, 0]^\top & r_3 = 1 \\ \bold{x}_4 = [1 \,\, 1]^\top & r_4 = 0 \end{matrix}x1​=[00]⊤x2​=[01]⊤x3​=[10]⊤x4​=[11]⊤​r1​=0r2​=1r3​=1r4​=0​

Fonctions de base​

  • Discriminant avec fonction de base
    • Transformation non linéaire ϕ:RD→RK\phi : \mathbb{R}^D \rightarrow \mathbb{R}^Kϕ:RD→RK traitée sous une forme linéaire
    • Généralement, nous aurons K>DK > DK>D
hi(x)−∑j=1Kwjϕi,j(x)+w0\text{h}_i(\bold{x}) - \sum_{j=1}^K w_j \phi_{i,j}(\bold{x}) + w_0hi​(x)−j=1∑K​wj​ϕi,j​(x)+w0​
  • Exemple de fonctions de base
    • ϕi,j(x)=xj\phi_{i,j}(\bold{x}) = x_jϕi,j​(x)=xj​
    • ϕi,j(x)=x1j−1\phi_{i,j}(\bold{x}) = x_1^{j-1}ϕi,j​(x)=x1j−1​
    • ϕi,j(x)=exp⁡(−(x2−mj)2/c)\phi_{i,j}(\bold{x}) = \exp(-(x_2-m_j)^2/c)ϕi,j​(x)=exp(−(x2​−mj​)2/c)
    • ϕi,j(x)=exp⁡(−∥x−mj∥2/c)\phi_{i,j}(\bold{x}) = \exp(-\|\bold{x}-\bold{m}_j\|^2/c)ϕi,j​(x)=exp(−∥x−mj​∥2/c)
    • ϕi,j(x)=sgn(xj−cj)\phi_{i,j}(\bold{x}) = \text{sgn}(x_j-c_j)ϕi,j​(x)=sgn(xj​−cj​)

Projection avec une fonction de base​

  • Transformation non linéaire ϕ:R1→R2\phi : \mathbb{R}^1 \rightarrow \mathbb{R}^2ϕ:R1→R2 traitée sous une forme linéaire
    • En 1D : non linéairement séparable
    • Avec projection en 2D : linéairement séparable

Fonctions de base​

  • Résolution du XOR avec une fonction de base ϕ:R2→R3\phi : \mathbb{R}^2 \rightarrow \mathbb{R}^3ϕ:R2→R3
ϕ(x)=[x1  x2  (x1x2)]⊤\phi(\bold{x}) = [x_1 \,\, x_2 \,\, (x_1x_2)]^\topϕ(x)=[x1​x2​(x1​x2​)]⊤
  • Résultats de la transformation
z1=[0  0  0]⊤r1=0z2=[0  1  0]⊤r2=1z3=[1  0  0]⊤r3=1z4=[1  1  1]⊤r4=0\begin{matrix} \bold{z}_1 = [0 \,\, 0 \,\, 0]^\top & r_1 = 0 \\ \bold{z}_2 = [0 \,\, 1 \,\, 0]^\top & r_2 = 1 \\ \bold{z}_3 = [1 \,\, 0 \,\, 0]^\top & r_3 = 1 \\ \bold{z}_4 = [1 \,\, 1 \,\, 1]^\top & r_4 = 0 \end{matrix}z1​=[000]⊤z2​=[010]⊤z3​=[100]⊤z4​=[111]⊤​r1​=0r2​=1r3​=1r4​=0​
  • Données linéairement sépoarables dans le nouvel espace !

Scikit-learn : modèles linéaires​

  • discriminant analysis.LinearDiscriminantAnalysis : méthodes paramétriques pour générer discriminants linéaires
