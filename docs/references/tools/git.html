Git et GitHub

Git est un système de gestion de versions distribué qui permet aux développeurs de suivre et de gérer les modifications de leur code source au fil du temps. Il facilite la collaboration en permettant à plusieurs personnes de travailler sur différents aspects d'un projet simultanément, sans écraser le travail des autres.

GitHub, quant à lui, est une plateforme en ligne qui utilise Git pour l'hébergement de projets de code source. Elle offre des outils supplémentaires pour la collaboration et l'intégration, tels que la gestion des "pull requests", l'hébergement de la documentation, et le suivi des problèmes.

Principales commandes Git​

  • git init : Initialisation d'un nouveau dépôt Git.
  • git clone <lien-du-repo> : Clonage d'un dépôt existant.
  • git add fichier1.txt fichier2.txt : Mise en attente des modifications pour le prochain commit.
  • git commit -m "Description" : Création d'un nouveau commit avec un message.
  • git push origin main : Envoi des commits vers un dépôt distant.
  • git fetch : Télécharge les mises à jour (commits, branches, tags, etc.) du dépôt distant vers votre dépôt local, mais ne modifie pas votre espace de travail. Cela permet de voir ce qui a été modifié dans le dépôt distant sans toucher à vos modifications locales. Une fois les mises à jour récupérées avec git fetch, nous pouvons fusionner ces modifications avec notre branche actuelle en utilisant git merge. En d'autres termes, git fetch permet de préparer le terrain pour un éventuel git merge.
  • git pull origin main : Récupération et fusion des modifications d'un dépôt distant. Équivalent à git fetch + git merge.
  • git branch nouvelle-branche : Gestion des branches (listage, création, suppression).
  • git checkout main : Changement de branche.
  • git merge nouvelle-branche : Fusion d'une branche dans une autre.
  • git status : Affichage du statut du dépôt.
  • git rebase : Réapplication des modifications d'une branche sur une autre.
  • git stash : Sauvegarde temporaire des modifications.
  • git revert <commit1>..<commit2> : Annulation des modifications d'un commit précédent.

Comment faire un Pull Request?​

Un "Pull Request" (souvent abrégé PR) permet à un contributeur de proposer des modifications à un projet, et à l'équipe du projet de les examiner, de les discuter et éventuellement de les fusionner.

Voici comment créer un Pull Request, en prenant l'exemple de GitHub :

  • Forkez le dépôt :
    • Allez sur la page du dépôt GitHub que vous souhaitez modifier.
    • Cliquez sur le bouton "Fork" en haut à droite. Cela va créer une copie du dépôt sur votre compte GitHub personnel.
  • Clonez votre fork :
    • Sur votre fork, cliquez sur le bouton "Code" pour obtenir l'URL de clonage.
    • Dans votre terminal ou interface Git, tapez : git clone [URL-de-votre-fork].
  • Créez une nouvelle branche :
    • Dans votre terminal, naviguez vers le répertoire du dépôt cloné : cd [nom-du-dépôt].
    • Créez une nouvelle branche : git checkout -b [nom-de-votre-branche].
  • Apportez vos modifications :
    • Modifiez ou ajoutez les fichiers nécessaires.
    • Utilisez git add [fichier] pour mettre en attente les modifications.
    • Utilisez git commit -m "Description des modifications" pour enregistrer vos modifications.
  • Poussez votre branche sur GitHub :
    • Tapez : git push origin [nom-de-votre-branche].
    • Exemple : git push origin alain.
  • Créez le Pull Request :
    • Retournez sur la page de votre fork sur GitHub.
    • Cliquez sur le bouton "New pull request".
    • Assurez-vous que la "base branch" est la branche où vous souhaitez que vos modifications soient fusionnées (souvent main ou master), et que la "compare branch" est celle que vous venez de pousser.
    • Complétez les informations du PR (description, issue associée si nécessaire, etc.).
    • Cliquez sur "Create pull request".
  • Attendez la revue :
    • Le ou les mainteneurs du projet vont examiner votre PR.
    • Ils peuvent demander des modifications, auquel cas vous pouvez les faire sur votre branche et pousser à nouveau : le PR sera automatiquement mis à jour.
    • Une fois que tout est approuvé, ils fusionneront votre PR dans le dépôt principal.
  • Synchronisez votre fork (optionnel, mais recommandé après chaque PR) :
    • Cela garantit que votre version du projet est à jour avec le dépôt principal.
    • Ajoutez le dépôt original comme une télécommande : git remote add upstream [URL-du-dépôt-original].
    • Récupérez les dernières modifications : git fetch upstream.
    • Basculez sur votre branche principale : git checkout main (ou git checkout master).
    • Fusionnez les modifications du dépôt principal : git merge upstream/main.
