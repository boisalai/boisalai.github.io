SAS

Created 2023-08-26. Last updated 2023-08-26.

SAS programs are composed of statements for data manipulation, analysis, and reporting. Programs are organized into steps, including DATA steps and PROC (procedure) steps.

Data Manipulation with DATA Step​

Use the DATA step to read, modify, and create datasets. Basic syntax:

DATA new_dataset_name;
    SET original_dataset_name;
    /* Data manipulation statements */
RUN;

Below is a more detailed example.

/* Assume we have an existing dataset named "original_data" */
data new_dataset; /* Creating a new dataset named "new_dataset" */
    set original_data; /* Reading data from "original_data" */
    
    /* Creating a new variable "total_amount" by summing two numeric variables */
    total_amount = sales_amount + tax_amount;
    
    /* Creating a categorical variable "region" based on a condition */
    if state = 'CA' then region = 'West';
    else if state in ('NY', 'NJ') then region = 'East';
    else region = 'Other';
    
    /* Subsetting data based on a condition */
    if year >= 2020; /* Keeping only data from the year 2020 and beyond */
run; /* End of the DATA step */

/* The new dataset "new_dataset" now contains the manipulated data */

Common instructions we can use within a DATA step​

  1. SET Statement: Reads data from an existing dataset into the current program data vector.
    set dataset_name;
  2. IF Statement: Conditionally processes observations based on specified conditions.
    if condition;
  3. WHERE Statement: Filters observations to include only those that meet specified conditions.
    `where condition;``
  4. DROP Statement: Excludes specific variables from the output dataset.
    drop variable_list;
  5. RENAME Statement: Renames variables in the output dataset.
    rename old_variable = new_variable;ailed exam
  6. KEEP Statement: Includes specific variables in the output dataset, excluding all others.
    keep variable_list;
  7. RETAIN Statement: Preserves values across iterations of the DATA step.
    retain variable_list;
  8. OUTPUT Statement: Writes observations to the output dataset.
    output;
  9. ARRAY Statement: Creates an array of variables to simplify repetitive processing.
    array array_name variable_list;
  10. DO Loop: Executes a series of statements repeatedly.
    do index_variable = start_value to end_value; /* Statements */ end;
  11. CALCULATED Keyword: Represents the value of a variable being calculated within an expression.
    new_variable = calculated old_variable * 2;
  12. Assignment Statements: Create or modify variables by assigning values or expressions.
    new_variable = expression;
  13. SUM Statement: Accumulates values for a variable.
    sum_variable + value;
  14. Character Functions: Manipulate character data using functions like SUBSTR, TRIM, COMPRESS, etc.
    new_string = substr(old_string, start, length);
  15. Numeric Functions: Perform mathematical operations on numeric data.
    new_value = sqrt(old_value);
  16. FORMAT Statement: Specifies formats for variables' appearance in the output.
    format variable_name format_name.;
  17. INFORMAT Statement: Specifies how data is read into variables from an external file.
    informat variable_name informat_name.;
  18. LABEL Statement: Adds descriptive labels to variables.
    label variable_name = 'Descriptive Label';
  19. LENGTH Statement: Specifies lengths for character variables.
    length character_variable $ length;

Filtering and Subsetting​

Use the WHERE statement to filter observations.

DATA filtered_data;
    SET original_data;
    WHERE condition;
RUN;

Creating New Variables​

Create new variables with assignment statements in the DATA step.

DATA new_data;
    SET old_data;
    new_variable = expression;
RUN;

Merging and Joining Datasets​

Combine datasets using the MERGE statement.

DATA merged_data;
    MERGE dataset1 dataset2;
    BY common_variable;
RUN;

Data Analysis with PROC Step​

Perform analyses and generate reports using PROC steps. Example: PROC MEANS for descriptive statistics.

PROC MEANS data=dataset_name;
    VAR numeric_variable;
    /* More options and statements */
RUN;

Commonly used PROC Step​

  1. PROC MEANS: Calculates basic descriptive statistics such as mean, median, minimum, maximum, and standard deviation for numeric variables.
  2. PROC SUMMARY: Similar to PROC MEANS, but with more advanced options for summarizing data.
  3. PROC FREQ: Generates frequency tables and statistics for categorical variables.
  4. PROC SORT: Sorts datasets based on one or more variables.
  5. PROC SQL: Executes SQL queries to perform data manipulation, joining, and subsetting.
  6. PROC PRINT: Prints selected observations and variables from a dataset for inspection.
  7. PROC CONTENTS: Displays metadata information about a dataset, such as variable names, types, and attributes.
  8. PROC REPORT: Generates customized tabular reports with advanced formatting options.
  9. PROC UNIVARIATE: Performs detailed univariate analysis, including histograms, normality tests, and more.
  10. PROC CORR: Calculates correlation coefficients between numeric variables.
  11. PROC REG: Performs linear regression analysis to model relationships between variables.
  12. PROC LOGISTIC: Conducts logistic regression for binary or multinomial outcomes.
  13. PROC ANOVA: Performs analysis of variance for comparing means across groups.
  14. PROC GLM: Performs general linear model analysis, including ANOVA and regression.
  15. PROC MIXED: Performs mixed-effects models for repeated measures or hierarchical data.
  16. PROC TRANSPOSE: Transposes data from long to wide format and vice versa.
  17. PROC SQL: Executes SQL queries within SAS to perform complex data manipulations and transformations.
  18. PROC IMPORT: Imports data from external files (CSV, Excel, etc.) into SAS datasets.
  19. PROC EXPORT: Exports SAS datasets to external files (CSV, Excel, etc.).
  20. PROC SGPLOT: Creates various types of statistical plots and graphs.
  21. PROC TABULATE: Generates highly customizable summary tables.

Generating Reports​

Use various PROCs for report generation (e.g., PROC PRINT, PROC REPORT).

PROC PRINT data=dataset_name;
    /* Options and statements */
RUN;

Advanced Analysis​

Use PROCs like PROC REG (regression), PROC GLM (general linear models) for advanced analysis. Example: PROC REG for linear regression.

PROC REG data=dataset_name;
    MODEL dependent_variable = independent_variables;
    /* More options and statements */
RUN;

Exporting Data​

Save datasets using the DATA step and OUTFILE statement.

DATA new_data;
    SET old_data;
    /* Data manipulation */
RUN;

PROC EXPORT data=new_data outfile='path\to\output.csv' dbms=CSV replace;
RUN;

Comments​

Use * for single-line comments and /* */ for multi-line comments.

* This is a single-line comment;
/* This is
   a multi-line
   comment; */

Macro​

In SAS, a macro is a way to generate and execute dynamic pieces of code. SAS macros are essentially pieces of code that can be defined, stored, and then invoked to perform repetitive tasks or to create flexible and customizable processes. Macros allow you to automate tasks, make your code more efficient, and enhance the reusability of your codebase.

Here's how macros work in SAS:

  1. Macro Definition: You create a macro by defining it using the %macro and %mend statements. Inside the macro definition, you can include parameters that are placeholders for values you'll provide when you invoke the macro.
%macro my_macro(parameter1, parameter2);
    /* Macro code using &parameter1 and &parameter2 */
%mend;
  1. Macro Invocation: To use the macro, you invoke it using the %macro_name syntax and provide values for the parameters.
%my_macro(value1, value2);
  1. Macro Variables: Inside a macro, you can reference parameters using the & prefix, like &parameter1. These are called macro variables. When you invoke the macro, the values you provide for the parameters are substituted in place of the macro variables.
  2. Dynamic Code Generation: Macros are especially useful for generating dynamic code. You can use macro variables to create code that adapts to different scenarios, datasets, variable names, and conditions.
  3. Conditional Logic and Loops: Macros can include conditional statements (%if, %else, %do) and loops (%do, %end) to create flexible and adaptive processes.
  4. Automatic Execution: When you invoke a macro, SAS replaces macro variables with their values and then executes the resulting code.
  5. %LET Statement: The %let statement is used to create macro variables that can be used throughout your SAS code. These variables can hold constants or dynamic values.
    %let data_path = "/path/to/data";

Below a more detailed example of a macro in SAS. In this example, we'll create a macro that generates a frequency report for a specified variable in a given dataset:

%macro frequency_report(dataset, variable);
    /* Creating a title for the report */
    title "Frequency Report for &variable in &dataset";

    /* Generating the frequency report using PROC FREQ */
    proc freq data=&dataset;
        tables &variable;
    run;

    /* Clearing the title after generating the report */
    title;
%mend;

/* Invoking the macro to generate a frequency report */
%frequency_report(sashelp.cars, make);
