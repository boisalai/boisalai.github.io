Julia

Démarrer avec Julia​

Pour démarrer Julia, exécutez la commande suivante :

exec '/Applications/Julia-1.7.app/Contents/Resources/julia/bin/julia'

Pour quitter la session interactive, utilisez Ctrl + D ou tapez exit() suivi de la touche Return ou Enter.

Pour créer un lien symbolique vers une version de Julia de votre choix (ici la version 1.7), utilisez le code suivant. Tapez julia dans votre shell et appuyez sur la touche Return pour lancer Julia.

sudo mkdir -p /usr/local/bin
sudo rm -f /usr/local/bin/julia
sudo ln -s /Applications/Julia-1.7.app/Contents/Resources/julia/bin/julia /usr/local/bin/julia

Vérifiez le lien avec :

which julia
# /usr/local/bin/julia
ls -l /usr/local/bin/julia*
# lrwxr-xr-x  1 utilisateur  admin  62  2 fév 20:10 /usr/local/bin/julia -> /Applications/Julia-1.7.app/Contents/Resources/julia/bin/julia
ls /Applications/Julia*/Contents/Resources/julia/bin/julia
# /Applications/Julia-1.7.app/Contents/Resources/julia/bin/julia

Ajouter le noyau Julia à Jupyter​

IJulia combine l'environnement interactif Jupyter avec le langage Julia.

Installez IJulia depuis le REPL Julia en appuyant sur ] pour entrer en mode pkg et en entrant :

julia> using Pkg
julia> Pkg.add("IJulia")
julia> Pkg.build("IJulia")

Ce processus installe également une spécification de noyau indiquant à Jupyter comment lancer Julia. Lancez ensuite le serveur notebook de la manière habituelle en exécutant jupyter notebook dans le terminal.

Pour plus d'informations, consultez IJulia.jl.

Julia dans Visual Studio Code​

Pour utiliser Julia dans Visual Studio Code :

  • Consultez Julia in Visual Studio Code.
  • Dans la palette de commandes de VS Code, choisissez > Julia: Start REPL.
  • Dans le Terminal, faites ] pour démarrer le "package manager", puis activate ..
  • Pour exécuter une seule instruction, faites Shift+Enter.
  • Pour exécuter le fichier, faites "Julia: Execute Active File in REPL".

Nouveautés de Julia 1.8​

Dans Julia 1.8, nous pouvons avoir des champs constants dans des structures mutables, donc la variable 'a' est mutable dans ce code mais 'b' est immuable.

mutable struct MyStruct
    a::Int
    const b::Int
end

Packages​

We can enter the package manager from the REPL by typing ].

To see all the possible commands in the REPL, simply enter Pkg mode by typing ] and then type ? followed by Return or Enter key.

(@v1.7) pkg> st
      Status `~/.julia/environments/v1.7/Project.toml` (empty project)

(@v1.7) pkg> add Flux
    Updating registry at `~/.julia/registries/General`
   Resolving package versions...
   Installed IRTools ────────────── v0.4.4
   Installed RealDot ────────────── v0.1.0
[...]

(@v1.7) pkg> st
      Status `~/.julia/environments/v1.7/Project.toml`
  [587475ba] Flux v0.12.8

(@v1.7) pkg> rm Flux
    Updating `~/.julia/environments/v1.7/Project.toml`
  [587475ba] - Flux v0.12.8
    Updating `~/.julia/environments/v1.7/Manifest.toml`
  [621f4979] - AbstractFFTs v1.0.1
  [1520ce14] - AbstractTrees v0.3.4
[...]

Type help to get help.

To exit the package manager session, use Ctrl + C.

We can also just do this.

julia> using Flux
 │ Package Flux not found, but a package named Flux is available from a
 │ registry.
 │ Install package?
 │   (@v1.7) pkg> add Flux
 └ (y/n) [y]:

PyCall.jl​

using PyCall

py_numpy = pyimport("numpy")

py"""

import numpy as np

def pi():
   return np.pi

"""

function area(pi_value, radius)
   return pi_value * radius^2
end

println("Multiple ways to access numpy.pi")
println(area(py"pi"(), 10))
println(area(py_numpy.pi, 10))
